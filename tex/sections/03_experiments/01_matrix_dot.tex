\subsection{01 matrix dot}
本小节报告矩阵列点积（\texttt{matrix\_dot}）实验结果，比较基线实现（\texttt{matrix\_dot\_naive}）与 cache 优化实现（\texttt{matrix\_dot\_cache}）在不同问题规模下的运行特征。本文统一采用 \texttt{mean\_ns\_per\_run} 作为主要性能指标，并结合归一化曲线与加速比热力图进行分析。

图 \ref{fig:matrix_normalized_curve} 显示，cache 优化在小规模区间收益有限，而在规模增大后相对优势逐步扩大。图 \ref{fig:matrix_speedup_heatmap} 进一步表明，tile 参数与问题规模存在显著耦合关系，错误的 tile 选择会削弱优化收益。

\input{tables/03_experiments/03_matrix_dot_results}

表 \ref{tab:matrix_dot_results} 对最优配置进行了量化汇总。总体上，最优 tile 从中小规模的 64/128 逐步收敛到大规模下的 256，对应加速比由约 $1.02\times$ 提升至约 $2.03\times$，说明 cache 局部性优化在大工作集场景下更具统计稳定性与工程价值。
